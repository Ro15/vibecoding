The method

A protocol, not a chat.

"Lead Architect mode: ON." Every session ran under the same three rules — so the work stayed disciplined and fully reproducible.

RULE 01

No manual edits

The AI provides all logic and every fix. I architect and direct; I never touch the code.

RULE 02

Full audit log

A prompts.md captures every prompt, every turn — the complete decision trail.

RULE 03

Time-boxed

An MVP target of 4–6h with elapsed-time checkpoints at the end of every response.

RULE 04

Verify before claiming

Nothing is "done" without fresh test evidence — output first, assertion second.

The architecture insight

One core. Three products.

All three projects are the same pipeline — ingest → normalize → pluggable analyzers → score/alert → database → API → dashboard — instantiating one shared, hexagonal core. Adding a cloud, a rule, a detector, or an alert channel is one new file with a decorator, zero engine edits. That's the open/closed principle turned into extension cost of O(1).

Project 1 · FinOps

CostOpt.

Ingests AWS / Azure / GCP / FOCUS billing exports, detects nine kinds of waste (unattached disks, idle & oversized VMs, orphaned IPs, old snapshots, idle load balancers…), and generates the exact CLI + SDK commands to decommission it — with a guarded, audited execution path.

Guardrail.

Project 3 · SRE

Watchdog.

Parses JSON / syslog / text logs and finds error-rate spikes with two detectors — an online EWMA + z-score (O(1) time & space per event, so memory stays bounded at any log volume) and a lightweight IsolationForest ML model — then fires simulated webhook alerts and animates health trends live.
